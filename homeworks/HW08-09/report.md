# HW08-09 – PyTorch MLP: регуляризация и оптимизация обучения

## 1. Кратко: что сделано

В работе обучался MLP‑классификатор на датасете **CIFAR10** (вариант C из `torchvision`).  
Датасет выбран, чтобы избежать проблем с загрузкой EMNIST и при этом потренировать все требуемые элементы пайплайна (train/val/test, регуляризация, оптимизация).

В части **A (S08)** сравнивались методы регуляризации:

- E1: базовая модель без регуляризации
- E2: Dropout
- E3: BatchNorm
- E4: EarlyStopping на лучшем конфиге из (E2/E3) и сохранение лучшей модели

В части **B (S09)** изучалось влияние оптимизации:

- O1: слишком большой learning rate
- O2: слишком маленький learning rate
- O3: SGD + momentum + weight decay

- **Устройство:** CPU

- **Как запустить:** открыть `HW08-09.ipynb` и выполнить *Run All*

## 3. Данные

- **Датасет:** CIFAR10  
- **Разделение:**  
  - train_full → train (80%) + val (20%)  
  - test — стандартный тестовый набор из torchvision  
- **Трансформации:** `ToTensor()` + `Normalize(mean, std)` (стандартные mean/std для CIFAR10)  
- **Комментарий:**  
  CIFAR10 содержит **10 классов**, изображения **RGB 3×32×32**.  
  Для MLP это сложный датасет, потому что модель не использует пространственную структуру изображения (в отличие от CNN).

---

## 4. Базовая модель и обучение

- **MLP:**  
  - вход: 3072 (3×32×32)  
  - скрытые слои: 256 → 256  
  - активация: ReLU  
  - опционально: Dropout / BatchNorm  
- **Loss:** CrossEntropyLoss  
- **Базовый оптимизатор (часть A):** Adam, lr=1e‑3  
- **Batch size:** 128  
- **Epochs (макс):** 20 для E1–E3, 50 для E4  
- **EarlyStopping:** patience=4, метрика — val_accuracy (минимальное улучшение 0.0005)  

---

## 5. Часть A (S08): регуляризация (E1–E4)

- **E1 (base):** 2 скрытых слоя, без Dropout/BatchNorm  
- **E2 (Dropout):** как E1 + Dropout(p=0.3)  
- **E3 (BatchNorm):** как E1 + BatchNorm  
- **E4 (EarlyStopping):** лучший из (E2/E3) + EarlyStopping (сохранение лучшего состояния по val_accuracy)  

---

## 6. Часть B (S09): LR, оптимизаторы, weight decay (O1–O3)

- **O1:** Adam, lr=1e‑1 (слишком большой)  
- **O2:** Adam, lr=1e‑5 (слишком маленький)  
- **O3:** SGD + momentum=0.9 + weight_decay=1e‑4, lr=1e‑2

## 8. Анализ

На CIFAR10 базовая MLP (E1) показывает ограниченное качество: задача сложная, а модель «не видит» структуру изображения, потому что работает с вектором из 3072 признаков.  
Тем не менее, сравнения регуляризации корректны: везде одинаковый split train/val и одинаковые базовые настройки обучения.

Dropout (E2) оказался лучшим из регуляризаторов по `best_val_accuracy`. Это ожидаемо: Dropout снижает склонность MLP к переобучению на небольших подвыборках и делает обучение более устойчивым.  
BatchNorm (E3) стабилизирует оптимизацию, но в моих результатах не дал прироста относительно Dropout по максимуму точности.

EarlyStopping в E4 остановил обучение на 14‑й эпохе (patience=4), сохранив лучший state_dict по `val_accuracy`. На этом датасете это полезно как «страховка» от ухудшения качества при продолжении обучения.

В части B:

- O1 (lr=1e‑1) — слишком большой шаг: качество остаётся близко к случайному для 10 классов (loss около `log(10)≈2.30`).  
- O2 (lr=1e‑5) — шаг слишком маленький: прогресс есть, но за 6 эпох модель не успевает существенно улучшиться.  
- O3 (SGD+momentum + weight decay) — обучается стабильно и по val_accuracy близок к лучшим прогонам Adam; weight decay выступает простой регуляризацией.

Выбранный лучший конфиг (Dropout=0.3 + Adam lr=1e‑3 + EarlyStopping) разумен для CIFAR10‑MLP: он простой, устойчивый и даёт воспроизводимый baseline, от которого дальше логично переходить к CNN и/или аугментациям.
